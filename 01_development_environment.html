Setting Up Your Development Environment

In order to write and test PHP scripts the appropriate software must be installed and configured:

  1. Apache Web Server + PHP + MySQL

    The dominant web server in the internet today is the open source Apache server. The PHP is a powerful open source scripting solution responsible for much online content delivery, including the main CMSs in use today.

    1. Download the lastest stable WAMP (Windows) or MAMP (Macintosh) server/php package.
    2. Install the package, taking the deafult settings as you go (be sure to install on a local hard drive (eg: C).
    3. When prompted, choose "Start Web Sever"
    4. The Wamp 'W' icon should appear in the lower right of the taskbar. Left clicking on this will reveal various options such as starting and stopping the server, and accessing the configuration files.
    5. Locate the www folder in your WAMP installation. This is your server root. All scripts must be located here or in subdirectories. (Macintosh users may have the server root folder named htdocs)
    6. We will secure your MySQL installation during a later session
    7. Test Your Web Server:
      1. use a web browser to vist http://localhost
      2. if a page displays, your server is working fine. if you get an error, ensure the 'W' icon is not in the red state
      3. troubleshooting: if a page does not display, you may need to change the default port of your HTTP server:
        • Confirm if port is the problem. Click the Wamp 'W' icon and choose Apace > Service > Test Port 80. If port 80 is being used by another application, proceed with the following steps:
        • Click the 'W' icon > Apache > httpd.conf
        • Find the 'Listen 80' section and change to 'Listen 8080'. Save the change
        • 'W' icon > Restart all services
        • Test your server using the new port: http://localhost:8080
      4. troubleshooting: if, when you test your localhost server, you get a message about not being authorized to access a remote server (a common error on laptops)
        • Instead of http://localhost use the literal localhost address http://127.0.0.1
    8. Test PHP:
      1. use a text editor (eg: notepad) to create a file named hello_world.php
      2. add the following in the file:
        <?php
	echo "Hello World!";
?>
      3. save the file
      4. test the file by using a web browser addressed http://localhost/hello_world.php
    9. Set the correct timezone for PHP to use by changing the configuration file. Use the 'W' icon to open php.ini. Find the 'date.timezone' declaration and make the following changes -
      date.timezone = America/Los_Angeles
      Save these changes and use the 'W' icon to restart all services.
    10. Macintosh Users Only: Ensure scripting errors are displayed by changing the configuration file. Locate and open the php.ini file. Find the 'display_errors' declaration and make the following changes, changin the 'Off' to 'On' -
      display_errors = On
      Save these changes and restart all services.
